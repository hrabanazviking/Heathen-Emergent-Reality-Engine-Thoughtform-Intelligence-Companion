# H.E.R.E.T.I.C. Codebase Structure Canon

This folder is the living structural map for the H.E.R.E.T.I.C. repository.

It is designed to do four jobs:

1. Make onboarding fast for humans and AI collaborators.
2. Preserve architectural intent as the project evolves.
3. Tie technical design directly to project culture and spiritual philosophy.
4. Keep system growth coherent across research, data, runtime, and implementation packs.

## Document Set

- `01_REPOSITORY_ATLAS.md` — high-level map of the current repository and major domains.
- `02_DIRECTORY_CONTRACTS.md` — purpose, allowed content, and maintenance rules for each major folder.
- `03_DOCUMENT_ARCHITECTURE_LAYERS.md` — conceptual architecture layers and how files map to each layer.
- `04_ENGINEERING_STANDARDS_AND_GUARDRAILS.md` — coding and design guardrails derived from project laws and rules.
- `05_CULTURE_ETHOS_AND_SYMBOLIC_LANGUAGE.md` — cultural operating model (Frith, reciprocity, anti-gatekeeping).
- `06_NORSE_HEATHEN_THIRD_PATH_ALIGNMENT.md` — how Norse Paganism, modern Viking culture, and the Heathen Third Path inform build decisions.
- `07_HERETIC_NAME_SEMANTIC_SPEC.md` — technical and cultural interpretation of the H.E.R.E.T.I.C. name.
- `08_GROWTH_ROADMAP_FOR_STRUCTURE.md` — staged plan for keeping structure healthy while the codebase scales.

## Maintenance Protocol

- Treat this folder as the source of truth for project structure.
- Update these files whenever a new major top-level directory, subsystem, or standards change is introduced.
- Prefer additive updates over destructive rewrites.
- Keep terminology synchronized with `README.md`, `RULES.AI.md`, and `PHILOSOPHY.md`.
